# ESG Sustainability Analytics — S&P 500 Companies

End-to-end ESG (Environmental, Social, Governance) analytics on a Kaggle dataset of 866 sustainability reports from 263 S&P 500 companies (2014-2023). The dataset comes pre-scored (text, entities and E/S/G scores are given); this project builds the analytics, machine-learning and GenAI layer on top of it — cleaning, validation, feature engineering, EDA, a company risk-index, a leakage-safe next-year score forecaster, a RAG copilot over the data, and an interactive dashboard.

Status: Steps 1-8 complete.

A full methodology write-up is in [REPORT.md](REPORT.md).

## Repository structure


The large CSVs (`preprocessed_content.csv`, `esg_features.csv`, `esg_risk.csv`) and the regenerated `data/` pickles are not committed. The input CSV is downloaded from Kaggle; the feature/risk artifacts are regenerated by the notebook.

## How to run

1. Download `preprocessed_content.csv` from Kaggle ("ESG Sustainability Reports of S&P 500 Companies") and place it next to the notebook.
2. Install dependencies: `pip install -r requirements.txt`
3. Run all cells in `ESG_Methodology_Steps_1-8.ipynb`. This regenerates the feature/risk artifacts, the figures, the trained model, and `data/features.pkl`.
4. **GenAI copilot (Step 7):** copy `.env.example` to `.env` and add a free Mistral key from [console.mistral.ai](https://console.mistral.ai), then run `python esg_copilot.py` (retrieval-only demo: `python esg_rag.py`, no key needed).
5. **Dashboard (Step 8):** `streamlit run app.py`

## Methodology summary

| Step | What | Result |
|------|------|--------|
| 1. Clean | drop index column, safe-parse entities, fix types, remove duplicate ticker+year | 864 x 9 |
| 2. Validate | verify total ≈ e+s+g, ranges, missing values | approximately additive (coeffs 1,1,1, R²=0.999), tolerance-checked |
| 3. Features | text, composition and leakage-safe lag features | 27 columns (18 engineered) |
| 4. EDA | 4 charts with insights | Social highest, Environmental lowest; disclosure rising over time |
| 5. Risk Index | 0.40 E + 0.30 S + 0.30 G, normalized and ranked | 263 companies ranked |
| 6. Forecast | leakage-safe regression | Linear Regression, R² 0.913, beats naive baseline |
| 7. GenAI Copilot | RAG over the ESG data (TF-IDF retrieval + Mistral generation) | data-grounded Q&A, key kept in `.env` |
| 8. Dashboard | Streamlit app (filters, KPIs, trend, top-10, table) | `streamlit run app.py`, demo-able |

## Key decisions

- No data leakage: the forecaster uses only past and structural features (shift(1) lag features). Current-year e/s/g/total and yoy_change are deliberately excluded, giving an honest R² of about 0.91 instead of a leakage-inflated 1.0.
- RAG for grounding: the copilot retrieves relevant reports (cosine similarity over TF-IDF vectors) and passes them as context, so the LLM answers from this dataset rather than from memory — reducing hallucination.
- Secrets stay out of git: the Mistral key is read from `.env` (git-ignored) via `os.getenv`, never hard-coded.
- Assumptions are verified, not assumed: the `total = e+s+g` relationship is only approximately true (about 22% of rows differ by more than rounding), so it is handled with a tolerance check and outlier flagging.
